# Eigenes Projekt anlegen

Die Website baut sich automatisch aus dem Repo. Du musst nur zwei Dinge anlegen.

## 1. Beispiel-Sketch in der Library

Die gesamte Logik steckt in der Library `lib/Touch/`. Lege ein neues Beispiel an:

```
lib/Touch/examples/meinBeispiel/meinBeispiel.ino
```

Editiere darin nur die Config-Tabelle:

```cpp
#include <TouchManager.h>

// { Pin, Event-Name, Taste, Threshold }
TouchConfig config[] = {
  {1, "links",  'a', 3000},
  {2, "rechts", 'd', 3000},
};

TouchManager touchManager;

void setup(){ touchManager.begin(config, MODE_KEYBOARD); }  // oder MODE_SERIAL
void loop(){ touchManager.update(); }
```

## 2. PlatformIO-Projekt, das es baut

Lege `firmware/meinBeispiel/` an mit zwei Dateien:

**`platformio.ini`**
```ini
[platformio]
src_dir = ../../lib/Touch/examples/meinBeispiel

[env:esp32-s3]
platform = espressif32
board = esp32-s3-devkitc-1
framework = arduino
lib_extra_dirs = ../../lib
build_flags = -DARDUINO_USB_MODE=1 -DARDUINO_USB_CDC_ON_BOOT=1
```

**`project.json`** (Metadaten für die Website)
```json
{
  "name": "Mein Beispiel",
  "slug": "meinBeispiel",
  "env": "esp32-s3",
  "chipFamily": "ESP32-S3",
  "mode": "hid",
  "description": "Was das Beispiel macht."
}
```

## 3. Pushen

Push auf `main` → die GitHub-Action baut die Firmware, erzeugt das
Flash-Manifest und veröffentlicht alles. Dein Beispiel erscheint **automatisch**
auf der [Flashen](/)-Seite – ohne die Website neu zu bauen.

> `chipFamily` muss zum `board` passen: `ESP32` (klassisch), `ESP32-S2`,
> `ESP32-S3` oder `ESP32-C3`. USB-HID (`mode: hid`) braucht S2/S3.
